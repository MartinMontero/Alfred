## ROLE

You are building **Alfred**, the sovereign, local-first, Nostr-native Personal Knowledge Management system for *agentic AI development* — a component of `wecanjustbuildthings.dev`. Alfred is the persistent external mind where a non-developer's specs, decisions, context, and memory live, and the ground truth their AI harness reads. The platform is the judgment layer; **goose** is the hands; **Alfred is the memory.** Alfred is an upgraded, independent evolution of the MIT-licensed `derekross/onyx` — **cloned (not forked)**, relicensed **AGPL-3.0-or-later**, re-identified as Alfred. The first build runs on **native Windows 11**.

You operationalize the research reports; you do **not** restate them. You work **one phase at a time**: do only the current phase, meet its "done when," write its audit note, commit, stop for review.

## MISSION & ETHOS (the why behind every rule below)

Alfred exists to give people who don't code the same external mind — judgment, memory, standards — that experienced builders carry, **without surrendering their data, their keys, or their autonomy**. Every decision serves four commitments, in priority order: **builder agency** (the person stays in control and informed), **security & privacy** (their knowledge and keys never leak), **sovereignty** (own the code, own the data, no dependence on extractive Big-AI platforms), and **openness** (AGPL, independent, auditable). When a choice is ambiguous, resolve it toward these — and tell the builder you did.

## BUILD TARGET: Windows 11 (native — no WSL2)

All toolchains are native Windows. See `docs/research/windows-build-2026-06.md` for the full facts. Load-bearing specifics:
- **Prereqs (already installed on this machine):** Git for Windows, Visual Studio Build Tools ("Desktop development with C++"), WebView2 (preinstalled), Rust via rustup (`stable-msvc`, triple `x86_64-pc-windows-msvc`), Node 22 LTS (≥ 22.12).
- **Build outputs:** `npm run tauri build` → raw `src-tauri\target\release\<app>.exe` (runs locally, unsigned) + installers in `src-tauri\target\release\bundle\{msi,nsis}\`. Defer code signing until distribution (tell the builder when that point arrives).
- **goose sidecar:** `src-tauri\binaries\goose-x86_64-pc-windows-msvc.exe` (the `.exe` is appended to the triple on Windows); `externalBin: ["binaries/goose"]`; spawn `app.shell().sidecar("goose")` (stem only); **kill the goose child on app exit.**
- **goose providers:** set `GOOSE_DISABLE_KEYRING=1` + env-var keys (Windows keyring is flaky); config at `%APPDATA%\Block\goose\config\config.yaml`.
- **Line endings:** commit a `.gitattributes` (`* text=auto eol=lf`, `*.exe binary`, mark `src-tauri/binaries/*` binary) — non-negotiable so the sidecar `.exe` isn't corrupted.
- **Shells:** PowerShell has no `grep` — use `Select-String` (or Git Bash). Use `cross-env` in npm scripts; audit `build`/`build:web` for bash-isms.

## NON-NEGOTIABLE CONSTRAINTS (the constitution — if a task seems to require breaking one, STOP and ASK)

1. **License:** ship AGPL-3.0-or-later. Keep upstream MIT notices; SPDX headers on every file; `ATTRIBUTION.md` (source repo, Derek Ross, exact import SHA, date, changes) and `NOTICE`. Record `nostr-tools` as **Unlicense** in SBOM/license docs.
2. **Vendor exclusion (absolute):** no dependency, import, endpoint, or config — direct or transitive — for **Meta, OpenAI, or xAI**. Google permitted. **React forbidden** (Meta-owned), incl. transitive: audit `qrcode.react`, replace if it pulls React. Base is SolidJS. Dogfood the platform engine: `npx tsx «PLATFORM_PATH»/enforcement/cli.ts all --tree .` is a blocking CI gate. **Stock goose compiles ALL providers in (no per-provider build flags — verified against goose v1.39.0 source; a "stripped" build would require a fork). The approved Phase-4 deviation (docs/audit/phase4.md) is a config-level lockdown: isolated `GOOSE_PATH_ROOT`, generated config, env lockdown with excluded-vendor keys blanked, ACP provider-list filtering — excluded providers are refused at Alfred's chokepoints, not removed from the binary.**
3. **ZERO Soapbox (absolute, hard rule).** No `@nostrify/nostrify`, no `@soapbox.pub/*` / `@soapbox/*` / `jsr:@soapbox/*`, nothing with `soapbox-pub` / Alex Gleason / MK Fain provenance, in deps **or** dev tooling (`.mcp.json`). Removal + migration to nostr-tools is Phase 1, **mandatory** (executed; applesauce not needed — see tooling table). Benchmark: `Select-String -Pattern soapbox` (or `grep -ri soapbox`) empty; `npm ls @nostrify/nostrify` empty; SBOM shows zero soapbox-pub provenance.
4. **Zero fabrication; primary sources only.** Don't invent APIs, versions, NIP/SEP numbers, or paths. Verify against the cloned source + official docs. If you can't verify, say so and stop.
5. **No unauthorized architectural decisions.** The phase plan is the scope. New deps, new Nostr event kinds, schema/rename changes need sign-off — open `docs/decisions/NNNN-*.md` and stop.
6. **Sovereignty/privacy by default.** Notes stay local, NIP-44-encrypted before any relay. Keys in the OS credential store / never logged. No telemetry without explicit opt-in. Treat vault + agent memory as an attack surface.
7. **Accessibility/quality are gates:** axe-core zero serious/critical WCAG 2.1 A/AA; Lighthouse perf ≥ 90, a11y ≥ 95, LCP ≤ 2.5 s, CLS ≤ 0.1 on the internal web-harness canonical screens (dev/test only — Alfred ships desktop-only; see ADR-0010).
8. **House style:** plain professional prose; **no emojis in code or UI unless requested**; BEM-like CSS; clean standalone files, not revision artifacts.
9. **Builder agency & transparency — no silent moves, no surprises.** The builder stays in control and informed at all times. Before any **environment change** (installing/updating tools, editing PATH, changing global config), any **destructive action** (deleting, overwriting, history rewrite, force-push), and any **outward action** (pushing to a remote, opening a PR, deploying, calling a network service, sending data off the machine), state plainly what you're about to do and why, and — unless it's a routine in-repo edit the current phase already authorizes — get the builder's go-ahead first. Never store, log, or transmit secrets or note content. Explain tradeoffs in plain language, surface risks early, record what you did in the phase audit note, and leave the final say with the builder. When in doubt, ask. A correct-but-unannounced action still violates this rule.

## DECISIONS (filled — each resolved toward builder agency, security/privacy, sovereignty, openness)

- **`«PLATFORM_PATH»`** → **`../wecanjustbuildthings.dev`** — the platform repo as a sibling folder; how Alfred reaches the platform's own exclusion engine to dogfood it. *(sovereignty: Alfred is policed by the project's own engine, not a third party.)*
- **`«ALFRED_REPO_URL»`** → **`https://github.com/MartinMontero/Alfred.git`** — the builder's own repo; all pushes go here. *(agency: the builder owns the code and its history.)*
- **Clone base** → **`derekross/onyx` `main` (v0.16.1) — clone the current `main`, then pin and record the exact commit SHA** in `UPSTREAM.md`. *(openness + security: a values-aligned MIT base; the pinned SHA makes provenance auditable and security back-ports traceable.)*
- **AI-provider denylist** → **exclude only Meta, OpenAI, and xAI; permit every other provider/model** (Anthropic, Google, Mistral, any open-source/open-weights provider, OpenRouter-routed open models, local via Ollama) — because none of them resolve to an excluded vendor. This is the **platform exclusion-engine model: a denylist enforced by vendor identity — *not* an allowlist of three**. Enforced app-side at the provider chokepoint and on the goose side by routing goose's provider/credential setup through the same denylist, with **env-var keys + `GOOSE_DISABLE_KEYRING=1`** on Windows (the goose mechanism is config-level; see `docs/audit/phase4.md`). *(sovereignty + privacy: no dependence on the excluded Big-AI vendors; keys live in env vars, never a flaky vault, never logged.)*
- **No hosted web app (builder's ruling 2026-07-23; supersedes the earlier "PWA deploy" decision) → ADR-0010.** Alfred is local-first: a native desktop application whose program and files live on the user's own machine. **There is NO user-facing PWA and no browser version — the concept is retired.** The `BUILD_TARGET=web` build (`dist-web/`, `vite-plugin-pwa`) survives **only** as an internal development/test harness (CI runs the axe-core + Lighthouse gates against it because this container can't build Tauri); it is **never deployed, never published, never shipped, and never positioned to a user.** The `alfred-pwa` Cloudflare Worker slot, if used at all, serves a **public explainer/landing page** for the project (Wayne Manor) that links to the desktop download — never the app itself. *(sovereignty/ethos: a hosted web app would store notes in browser OPFS rather than as plain files on the user's disk — the opposite of the local-first promise.)*
- **Deep-link scheme / bundle id** → **`alfred://`** and **`dev.wecanjustbuildthings.alfred`** — stable OS-level identifiers for the app. *(agency: a clear, collision-proof identity the builder controls.)*
- **License** → **AGPL-3.0-or-later** — strong copyleft; anything built on Alfred stays open. *(openness: matches the project's whole reason for being.)*
- **Clone-not-fork** → **accepted**, with an `UPSTREAM.md` recording the source + pinned SHA and a documented **security cherry-pick cadence** (periodically review upstream for security fixes and port just those). *(sovereignty + security: full independence from upstream governance, with a deliberate plan so independence doesn't mean missing security fixes — and the builder is told when a back-port is pulled in.)*
- **Local folder name** → **`Alfred`** (capital A), matching the GitHub repo, so folder, repo, and push commands all line up. *(agency: no confusion between local and remote.)*
